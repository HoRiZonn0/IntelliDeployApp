# 1. 接口 A：启动降级生成任务

## 用途

当出现以下情况时，调用杨的生成模块：

- 候选仓库都不合适
- Top1 修复失败，转 Branch B
- 人工指定强制走生成

---

## A-1 请求字段

| 字段                                     | 类型     | 必填 | 说明 / 取值                                                                            |
| ---------------------------------------- | -------- | ---: | -------------------------------------------------------------------------------------- |
| `project_id`                           | string   |   是 | 业务项目 ID                                                                            |
| `deployment_id`                        | string   |   是 | 本次部署记录 ID                                                                        |
| `request_id`                           | string   |   否 | 链路追踪 ID，便于日志排查                                                              |
| `trigger_reason`                       | enum     |   是 | `LOW_SCORE_ALL` / `REPAIR_EXHAUSTED` / `FORCE_FALLBACK`                          |
| `original_prompt`                      | string   |   是 | 用户原始需求                                                                           |
| `generation_mode`                      | enum     |   是 | `AUTO` / `VIBE` / `COMPONENT_REASSEMBLY`                                         |
| `evaluation_score`                     | integer  |   否 | 评估分数，0-100                                                                        |
| `missing_components`                   | string[] |   否 | 缺失组件列表，如 `["Dockerfile", "start_script"]`                                    |
| `preferred_stack.frontend`             | string   |   否 | 如 `React` / `Next.js`                                                             |
| `preferred_stack.backend`              | string   |   否 | 如 `FastAPI` / `Node.js`                                                           |
| `preferred_stack.database`             | string   |   否 | 如 `PostgreSQL` / `SQLite`                                                         |
| `preferred_stack.runtime`              | string   |   否 | 如 `python3.11` / `node18`                                                         |
| `repo_profile.source_repo_url`         | string   |   否 | 原仓库地址；如果是纯生成可为空                                                         |
| `repo_profile.detected_languages`      | string[] |   否 | 检测出的语言                                                                           |
| `repo_profile.detected_frameworks`     | string[] |   否 | 检测出的框架                                                                           |
| `repo_profile.package_manager`         | enum     |   否 | `npm` / `pnpm` / `yarn` / `pip` / `poetry` / `go` / `maven` / `gradle` |
| `repo_profile.entrypoints`             | string[] |   否 | 如 `["main.py", "app.js"]`                                                           |
| `repo_profile.dependency_files`        | string[] |   否 | 如 `["package.json", "requirements.txt"]`                                            |
| `repo_profile.has_valid_dockerfile`    | boolean  |   否 | 是否已有可疑似复用 Dockerfile                                                          |
| `repo_profile.readme_summary`          | string   |   否 | 你提取后的仓库摘要                                                                     |
| `constraints.timeout_seconds`          | integer  |   否 | 单次生成超时限制                                                                       |
| `constraints.target_port`              | integer  |   否 | 希望服务暴露的端口，如 `80/8080/3000`                                                |
| `constraints.must_provide_dockerfile`  | boolean  |   否 | 默认建议 `true`                                                                      |
| `constraints.must_provide_healthcheck` | boolean  |   否 | 默认建议 `true`                                                                      |

---

## A-2 响应字段

| 字段          | 类型            | 必填 | 说明                        |
| ------------- | --------------- | ---: | --------------------------- |
| `accepted`  | boolean         |   是 | 是否已接收任务              |
| `task_id`   | string          |   是 | 异步任务 ID                 |
| `status`    | enum            |   是 | 初始值建议固定为 `QUEUED` |
| `queued_at` | datetime string |   是 | 入队时间                    |
| `message`   | string          |   否 | 简短说明                    |

---

# 2. 接口 B：查询生成任务状态

## 用途

轮询杨的任务状态，或者杨提供状态查询接口以调用。

---

## B-1 请求字段

| 字段        | 类型   | 必填 | 说明           |
| ----------- | ------ | ---: | -------------- |
| `task_id` | string |   是 | 异步任务 ID |

---

## B-2 响应字段

| 字段                 | 类型            | 必填 | 说明 / 取值                                                                                            |
| -------------------- | --------------- | ---: | ------------------------------------------------------------------------------------------------------ |
| `task_id`          | string          |   是 | 任务 ID                                                                                                |
| `project_id`       | string          |   是 | 业务项目 ID                                                                                            |
| `deployment_id`    | string          |   是 | 部署记录 ID                                                                                            |
| `status`           | enum            |   是 | `QUEUED` / `RUNNING` / `SUCCEEDED` / `FAILED`                                                          |
| `current_stage`    | enum            |   否 | `queued` / `classifying` / `solving` / `materializing` / `validating` / `packaging` / `completed` / `failed` / `manual_required` |
| `progress_message` | string          |   否 | 前端可展示的短描述                                                                                     |
| `artifact_ready`   | boolean         |   是 | 是否已可拉取结果                                                                                       |
| `updated_at`       | datetime string |   是 | 最近更新时间                                                                                           |
| `error_code`       | enum            |   否 | 失败时返回                                                                                             |
| `error_message`    | string          |   否 | 失败摘要                                                                                               |
| `recoverable`      | boolean         |   否 | 是否可重试                                                                                             |

说明：

- `status` 只表示任务主状态，不再承载业务阶段。
- 旧的 `GENERATING` / `STITCHING` / `PACKAGING` 如需兼容，建议在网关层映射，不再作为内部主状态继续扩散。

---

# 3. 接口 C：获取生成产物结果

## 用途

当 `status = SUCCEEDED` ，需要拿到杨生成的“可部署产物”。

以下三种方式选一

- `artifact_path`：本地工作目录路径（我觉得可以这种）
- `artifact_uri`：对象存储/压缩包地址
- `artifact_key`：统一存储系统中的 key

---

## C-1 请求字段

| 字段        | 类型   | 必填 | 说明    |
| ----------- | ------ | ---: | ------- |
| `task_id` | string |   是 | 任务 ID |

---

## C-2 响应字段

| 字段                         | 类型     | 必填 | 说明 / 取值                                 |
| ---------------------------- | -------- | ---: | ------------------------------------------- |
| `task_id`                  | string   |   是 | 任务 ID                                     |
| `artifact_type`            | enum     |   是 | `TEMPLATE_PROJECT` / `STITCHED_PROJECT` |
| `artifact_path`            | string   |   否 | 本地路径                                    |
| `artifact_uri`             | string   |   否 | 远端产物地址                                |
| `artifact_key`             | string   |   否 | 存储键                                      |
| `dockerfile_content`       | string   |   是 | 最终 Dockerfile                             |
| `runtime.base_image`       | string   |   否 | 如 `python:3.11-slim`                     |
| `runtime.package_manager`  | string   |   否 | 如 `pip` / `pnpm`                       |
| `runtime.install_command`  | string   |   否 | 安装命令                                    |
| `runtime.start_command`    | string   |   是 | 启动命令                                    |
| `runtime.exposed_port`     | integer  |   是 | 对外暴露端口                                |
| `runtime.healthcheck_path` | string   |   否 | 如 `/health`                              |
| `required_envs`            | object[] |   是 | 见下方子结构                                |
| `warnings`                 | string[] |   否 | 非阻断性警告                                |
| `summary`                  | string   |   否 | 生成摘要                                    |
| `deploy_ready`             | boolean  |   是 | 是否可直接进入部署                          |
| `next_action`              | enum     |   否 | `DEPLOY` / `MANUAL_REVIEW`              |

---

## C-3 `required_envs` 子结构

这个一定要结构化，避免“幻觉环境变量”问题。

| 字段              | 类型    | 必填 | 说明                       |
| ----------------- | ------- | ---: | -------------------------- |
| `name`          | string  |   是 | 环境变量名                 |
| `required`      | boolean |   是 | 是否必填                   |
| `example_value` | string  |   否 | 示例值                     |
| `description`   | string  |   否 | 用途说明                   |
| `source`        | enum    |   否 | `DETECTED` / `ASSUMED` |

建议约定：

- `source=ASSUMED` 的变量默认不能直接进入部署

---

# 4. 接口 D：部署失败后回传修复/重生成请求

## 用途

当部署到 Sealos 后，如果失败，需要把**清洗后的错误信息**回传给杨，决定是否重新生成。

---

## D-1 请求字段

| 字段                                    | 类型    | 必填 | 说明 / 取值                                                               |
| --------------------------------------- | ------- | ---: | ------------------------------------------------------------------------- |
| `project_id`                          | string  |   是 | 项目 ID                                                                   |
| `deployment_id`                       | string  |   是 | 部署记录 ID                                                               |
| `source_task_id`                      | string  |   是 | 原生成任务 ID                                                             |
| `failed_stage`                        | enum    |   是 | `BUILD` / `RUN` / `HEALTHCHECK`                                     |
| `error_type`                          | enum    |   否 | 如 `DEPENDENCY_MISSING` / `PORT_MISMATCH` / `START_COMMAND_INVALID` |
| `sanitized_error_log`                 | string  |   是 | 你清洗后的错误摘要                                                        |
| `last_dockerfile_content`             | string  |   是 | 当前失败用的 Dockerfile                                                   |
| `healthcheck_result.status_code`      | integer |   否 | 健康检查返回码                                                            |
| `healthcheck_result.response_snippet` | string  |   否 | 响应摘要                                                                  |
| `retry_count`                         | integer |   是 | 已重试次数                                                                |
| `regen_mode`                          | enum    |   是 | `PATCH_EXISTING` / `REGENERATE`                                       |
| `constraints.timeout_seconds`         | integer |   否 | 本轮限制                                                                  |

---

## D-2 响应字段

| 字段         | 类型    | 必填 | 说明                      |
| ------------ | ------- | ---: | ------------------------- |
| `accepted` | boolean |   是 | 是否接受修复任务          |
| `task_id`  | string  |   是 | 新任务 ID                 |
| `status`   | enum    |   是 | 初始状态，建议 `QUEUED` |
| `message`  | string  |   否 | 说明信息                  |
